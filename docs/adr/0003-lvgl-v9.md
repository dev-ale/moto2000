# ADR 0003: LVGL v9 as the UI library

Date: 2026-04-09
Status: Accepted

## Context

The ESP32 renders every dashboard screen on a 466×466 round AMOLED. We need smooth
animations, efficient partial redraws, round-display primitives, and a host simulator
for CI snapshot tests.

## Decision

Use **LVGL v9.2** as the UI library, as an ESP-IDF managed component. Use the
LVGL SDL host simulator for CI snapshot tests.

## Consequences

Positive:

- Mature, well-documented, actively maintained
- First-class round-display support (arcs, circular scales, rotation)
- SDL host simulator enables snapshot tests on Linux CI without hardware
- Efficient partial redraws suit AMOLED power characteristics
- Large widget library (gauges, graphs, labels) covers most PRD screens

Negative:

- Memory footprint is non-trivial; must tune buffers for the 466×466 display
- v9 is a breaking change from v8; community examples sometimes lag

Rejected alternatives:

- **Custom blitter** — too much work for no benefit
- **TFT_eSPI / Arduino graphics** — not available under ESP-IDF without porting;
  weaker widget library
